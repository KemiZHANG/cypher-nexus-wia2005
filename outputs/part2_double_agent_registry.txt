Part 2 - The Double Agent Registry

Mission Problem
Detect suspicious identities using exact duplicate evidence and near-duplicate alias evidence.

Algorithm Used
Two-Stage Hybrid Verification System: Hash Table Filtering + Levenshtein Distance

Rejected Algorithms
- Linear search is rejected because repeated pairwise checking is too slow for large registries.
- Hash table only is rejected because it cannot detect near-duplicate aliases such as Falcon and Falcon_1.

Result Explanation
Hash tables quickly identify exact repeated evidence, then Levenshtein distance adds near-alias evidence before ranking identities by suspicion score.

Time and Space Complexity
Hash filtering O(n); near-alias verification O(a^2*m*n) for unique aliases and string lengths; space O(n).
